## Virtual Reality in the Web
VR is a new media.
For VR to take off, we need **content** and content requires authors and distribution.
The Web and Web technologies are free, frictionless and reach millions. They seems the [perfect choice for such a mission](../web-platform-when-why/).
WebVR is an initiative started by Mozilla for bringing VR experiences to the Web.
[Initial implementations](http://web.archive.org/web/20151016091437/http://blog.bitops.com/blog/2014/06/26/first-steps-for-vr-on-the-web) were a hack on the Fullscreen API in Firefox and Chrome experimental versions.
These experiments lead the to [WebVR 1.1](https://w3c.github.io/webvr/spec/1.1/).
The [WebVR sepecficiation](https://github.com/w3c/webvr/issues) is edited by the [Mozilla MozVR team](https://mozvr.com/), Google and Microsoft.
Apple, Samsung, Oculus and others also participate in the [**open** discussions](https://github.com/w3c/webvr/issues).
WebVR **enables access to VR devices**, including head-mounted displays, sensors and controllers.
Browsers supporting WebVR: [Firefox Nightly, Chromium, Servo, Chrome for Android, Samsung Internet, Carmel, MS Edge](https://webvr.rocks/)
So now we have a Web standard, it should be as easy as...
* Getting the VR display * Creating a WebGL 3D context * Getting pose of the display * Getting the eye parameters * Drawing the scene (once per eye) and…
* Providing polyfills * Dealing with fullscreen * Dealing with mobile * Dealing with resize …

...oh, WAIT!

## A-Frame HTML for Virtual Reality
A-Frame is a web framework for **building VR experiences**. It drastically reduces the boilerplate to:
```html <a-scene></a-scene> ```

A 360 picture viewer are 6 lines of HTML code.

See the Pen 360º panorama viewer by Salvador de la Puente González (@lodr) on CodePen.

A-Frame uses [Custom Elements](http://jonrimmer.github.io/are-we-componentized-yet/) to extend the HTML vocabulary enabling the use of new **tags for the 3D scene management** such as `a-scene`, `a-box` or `a-camera`.

A-Frame provides general primitives for building scenes: * [`a-scene`](https://aframe.io/docs/0.6.0/core/scene.html) is the root of the 3D scene. * [`a-animation`](https://aframe.io/docs/0.6.0/core/animations.html) allows animation of scene elements. * [`a-sky`](https://aframe.io/docs/0.6.0/primitives/a-sky.html) for setting a skydome. * [`a-box`](https://aframe.io/docs/0.6.0/primitives/a-box.html), `a-plane`, `a-sphere`... to quicky setup geometric primitives.
However, **`a-entity` elements and _components_** are the real building blocks of A-Frame.

## Entities, Components & Systems
The real power of A-Frame lies in its [ECS architecture](https://aframe.io/docs/0.6.0/core/).
### E is for Entity An [entitiy](https://aframe.io/docs/0.6.0/core/entity.html) is an element of the scene and a container for components. ```html <a-entity></a-entity> ```
### C is for Component A [component](https://aframe.io/docs/0.6.0/core/component.html) captures an aspect of an entity. ``` <a-entity position="0 2 -5"></a-entity> ``` Review the [complete list of components](https://aframe.io/docs/0.6.0/components/)!
Components can store multiple data using a sort of CSS syntax. ```html <a-entity position="0 2 -5" camera="near: 0.1; far: 100"></a-entity> ```
### S is for System A [system](https://aframe.io/docs/0.6.0/core/systems.html) is the logic governing entities possesing a component of the same aspect.
The ECS pattern is **the source of A-Frame flexibility**.
Even primitives are actually entities whose attributes map to specific components.
So, the primitive: ```html <a-box color="red"></a-box> ``` is actually a shortcut for: ```html <a-entity geometry="primitive: box" material="color: red"></a-entity> ```
## The A-Frame registry
Components are the main point of extension in A-Frame. Some are built-in in A-Frame:
`camera`, `cursor`, `debug`, `embedded`, `geometry`, `light`, `position`, `rotation`, `scale`, `sound`, `shadow`, `hand-controls`, `gltf-model`...
But developers can create their own components:
`environment`, `alongpath`, `animation`, `mountain`, `outline`, `particle-system`, `physics`, `teleport-controls`, `auto-detect-controllers`, `bmfont-text`, `crawling-cursor`, `firebase`, `layout`, `speech-command`, `stats-in-vr`, `stereo`, `super-hands`...
Find them all in the [A-Frame Registry](https://aframe.io/aframe-registry/)!
Using a 3rd party component is as simple as including the component file. ```html <script src="https://aframe.io/releases/0.6.1/aframe.min.js"></script> <script src="https://rawgit.com/feiss/aframe-environment-component/master/dist/aframe-environment-component.min.js"></script> ```
